# Развертывание Drone CI с использованием Docker Compose

## Описание

Данный файл `docker-compose.yml` предназначен для развертывания собственного экземпляра Drone CI — системы непрерывной интеграции и доставки. В состав входят два сервиса: сервер Drone и агент для выполнения пайплайнов. Решение позволяет организовать CI/CD процесс для проектов, размещенных на GitHub, без использования облачных версий Drone.

## Используемые технологии

- **Drone Server** — центральный компонент, управляющий репозиториями, пайплайнами и аутентификацией
- **Drone Runner Docker** — агент, выполняющий пайплайны в Docker-контейнерах
- **Docker** — контейнеризация всех компонентов системы
- **Docker Compose** — оркестрация сервисов Drone

## Структура конфигурации

```
services:
  drone-server:      # Серверная часть Drone
  drone-agent:       # Агент для выполнения пайплайнов
volumes:
  ./drone-data:      # Хранилище данных Drone
  /var/run/docker.sock: # Docker-сокет для управления контейнерами
```

## Конфигурация сервисов

### Drone Server (`drone-server`)

**Образ:** `drone/drone:2`  
**Порт:** `80:80` (HTTP-интерфейс и API)  
**Тома:**
- `./drone-data:/data` — постоянное хранилище данных

**Переменные окружения:**

| Переменная | Описание |
|------------|----------|
| `DRONE_GITHUB_CLIENT_ID` | Client ID OAuth приложения GitHub |
| `DRONE_GITHUB_CLIENT_SECRET` | Client Secret OAuth приложения GitHub |
| `DRONE_RPC_SECRET` | Секретный ключ для RPC-коммуникации между сервером и агентом |
| `DRONE_SERVER_HOST` | Хост или IP-адрес, по которому доступен Drone Server |
| `DRONE_SERVER_PROTO` | Протокол (http/https) |
| `DRONE_USER_CREATE` | Создание пользователя-администратора при первом запуске |
| `DRONE_OPEN` | Открытая регистрация новых пользователей |
| `DRONE_DOCKER_ENABLED` | Включение поддержки Docker-плагинов |

### Drone Agent (`drone-agent`)

**Образ:** `drone/drone-runner-docker:1`  
**Зависимости:** `depends_on: drone-server`  
**Тома:**
- `/var/run/docker.sock:/var/run/docker.sock` — доступ к Docker-сокету хоста

**Переменные окружения:**

| Переменная | Описание |
|------------|----------|
| `DRONE_RPC_PROTO` | Протокол для подключения к серверу |
| `DRONE_RPC_HOST` | Адрес сервера (внутреннее имя сервиса) |
| `DRONE_RPC_SECRET` | Секретный ключ для RPC-коммуникации (должен совпадать с серверным) |
| `DRONE_RUNNER_CAPACITY` | Количество одновременно выполняемых пайплайнов |

## Предварительные требования

Перед запуском системы необходимо:

1. **Зарегистрировать OAuth приложение в GitHub**
   - Перейти в Settings → Developer settings → OAuth Apps → New OAuth App
   - Homepage URL: `http://<ваш-ip-или-домен>`
   - Authorization callback URL: `http://<ваш-ip-или-домен>/login`
   - Получить Client ID и Client Secret

2. **Сгенерировать RPC секрет**
   ```bash
   openssl rand -base64 16
   ```

3. **Убедиться, что Docker установлен** на хост-системе

4. **Проверить доступность порта 80** на хосте

## Запуск

### Настройка переменных окружения

Замените следующие значения в файле `docker-compose.yml`:

```yaml
environment:
  - DRONE_GITHUB_CLIENT_ID=ваш_client_id
  - DRONE_GITHUB_CLIENT_SECRET=ваш_client_secret
  - DRONE_RPC_SECRET=сгенерированная_строка
  - DRONE_SERVER_HOST=ваш_ip_или_домен
  - DRONE_USER_CREATE=username:ваш_github_username,admin:true
```

### Запуск системы

```bash
docker compose up -d
```

### Проверка работоспособности

Откройте в браузере: `http://<ваш-ip-или-домен>`

Должна открыться страница входа через GitHub.

## Настройка пайплайна

После успешного запуска Drone:
1. Авторизуйтесь через GitHub
2. Активируйте нужные репозитории через интерфейс Drone
3. Создайте в корне репозитория файл `.drone.yml` с описанием пайплайна
4. При push-коммите в репозиторий Drone автоматически запустит пайплайн

## Важненькое

- **Данные**: Папка `./drone-data` содержит базу данных и конфигурацию Drone. Рекомендуется регулярно создавать резервные копии.
- **Docker сокет**: Агент использует Docker-сокет хоста для выполнения пайплайнов. Убедитесь, что пользователь, от которого запускается Docker, имеет права доступа к `/var/run/docker.sock`.
- **Безопасность**: Для production-среды используйте HTTPS вместо HTTP. Настройте переменную `DRONE_SERVER_PROTO=https` и используйте обратный прокси (например, Nginx) с SSL-сертификатом.
- **Лимиты**: Переменная `DRONE_RUNNER_CAPACITY` определяет количество параллельно выполняемых пайплайнов. Увеличивайте значение с учетом ресурсов сервера.
